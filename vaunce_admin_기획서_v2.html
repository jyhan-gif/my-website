01 · OVERVIEW

기획 배경 및 목적

BACKGROUND

현재 운영 메뉴에서 필터 조회 후 전체 합계(금액·건수·횟수 등)를 확인하려면 엑셀 다운로드 후 별도 집계가 필요합니다. 운영 담당자의 업무 효율이 저하되고 있습니다.

GOAL

필터 조회 결과에 대한 전체 합계를 테이블 내 고정 합계 행으로 즉시 표시합니다. 엑셀 없이 한 화면에서 의사결정이 가능하도록 개선합니다.

적용 범위운영 > 결제 내역 / 다회권 내역 / 정기권 내역 / 일회권 내역 / 포인트 관리 (총 5개 페이지)
UI 변경 범위기존 테이블 컬럼 구조 변경 없음. 컬럼 헤더 바로 아래 합계 행(tr) 1개 추가만 해당
합계 기준페이지(20건) 기준이 아닌 필터 조회 전체 결과 기준으로 합산

06 · PRIORITY

우선순위 제안

P1

결제 내역 / 다회권 내역

조회 빈도 가장 높고, 금액 집계 필요성이 즉각적.

P2

일회권 내역 / 정기권 내역

다회권과 구조 유사 — 컴포넌트 재사용 가능.

P3

포인트 관리

컬럼 구조 상이 — 별도 집계 로직 구현 필요.

본 문서의 수치는 기획용 예시 데이터이며, 실제 집계 값은 운영 DB 기준으로 산출됩니다.
